# 高速公路收费站调度模拟器 — 完整设计文档

## 这是什么

一个单页 HTML 网页游戏（纯前端，不需要后端），灵感来自 [airport.apunen.com](https://airport.apunen.com/) 的机场调度模拟器。

核心体验：玩家是收费站的调度员，车辆从公路两端不断涌入，你要实时给每辆车规划路线，引导它们通过正确的收费口，不能让车与车相撞，也不能让车堵在路上太久。

**最终产物：一个可以直接在浏览器打开的单 HTML 文件，内联所有 CSS 和 JS，不依赖任何外部库。**

---

## 为什么好玩 —— 你必须理解这些才能做出好玩的游戏

这不是一个"能跑就行"的技术 Demo。这是一个**让人停不下来**的游戏。以下五点是游戏好玩的根本原因，缺一不可：

1. **车辆一直在动**：车不会等玩家。玩家不操作，车就按自己的方向继续开，可能撞上别的车或者开出屏幕。这是紧张感的来源——"不操作 = 出事"。

2. **冲突自然产生**：从上方来的车要去下方出口，从下方来的车要去上方出口，路线天然会在收费广场交叉。玩家必须错开时间或者改变路线来避免碰撞。不是系统制造危机，是车辆运动自然演化出冲突。

3. **节奏递增**：开始时车很少（每 5 秒来一辆），玩家从容应对、学会操作；随着时间推移，车越来越多（每 2 秒来一辆）、速度逐渐加快，同时在场的车辆数量增加。1 分钟后开始紧张，2 分钟后手忙脚乱。

4. **操作极简但决策不简单**：玩家唯一的操作是"给车画路线"（点击/触摸车辆，拖动画出路径，松手确认，车辆沿路径行驶）。操作三秒就能学会，但"先处理哪辆车、路线怎么画才不跟别的车冲突"需要实时判断。

5. **失败清晰、想再来一局**：两车相撞 = Game Over，碰撞瞬间有视觉爆炸反馈。失败后显示成绩和"再来一局"按钮。好的调度游戏让人死了之后想说"再来一次我肯定能更好"。

## 画面布局（俯视图）

```
          【上方入口：3 个车道，车辆从这里驶入】
                ↓     ↓     ↓
   ┌──────────────────────────────────────┐
   │                                      │
   │         上方缓冲区（变道区域）          │
   │                                      │
   ├──────────────────────────────────────┤
   │                                      │
   │    [口1]  [口2]  [口3]  [口4]  [口5]  │
   │     ▏      ▏      ▏      ▏      ▏    │
   │    收费口（车辆在此停留 2-4 秒）        │
   │                                      │
   ├──────────────────────────────────────┤
   │                                      │
   │         下方缓冲区（变道区域）          │
   │                                      │
   └──────────────────────────────────────┘
                ↓     ↓     ↓
          【下方出口：3 个车道，车辆从这里驶出】
```

### 布局说明

- **上方入口**：3 条车道，车辆从屏幕顶部生成并向下行驶
- **下方入口**：3 条车道，车辆从屏幕底部生成并向上行驶（从高速下来的车）
- **收费口**：横向排列 5 个，位于画面中央
- **缓冲区/收费广场**：收费口上下各有一片开阔区域，车辆可以在这里自由变道和绕行——这是玩家画路线的主要操作区域
- 从上方进入的车，通过收费口后从下方出口离开
- 从下方进入的车，通过收费口后从上方出口离开
- **双向车流在收费广场交汇，路线天然交叉——这就是冲突和紧张感的来源**

## 核心机制

### 一、车辆

| 类型 | 颜色 | 速度 | 体积 | 特殊规则 |
|------|------|------|------|---------|
| 小轿车 | 蓝色 | 快 | 小 | 无 |
| 货车 | 黄色 | 慢 | 大（占 1.5 倍宽度） | 转弯半径大，路线不能太急 |
| 特种车（救护车/警车） | 红色闪烁 | 中 | 小 | 必须在 10 秒内通过收费口，否则扣分 |

- 每辆车生成时带有一个**目标方向标记**（小箭头或闪烁的出口提示），告诉玩家这辆车要去哪个方向
- **如果玩家不给车画路线，车会沿直线前进**——大概率撞上对面来的车或者堵在收费口前面
- 车辆之间有碰撞体积，不能重叠

### 二、路线规划（核心交互）

这是玩家唯一的操作方式，必须做到丝滑、直觉、有反馈：

1. **点击/触摸一辆车** → 车辆高亮，进入"画线模式"
2. **按住并拖动** → 手指/鼠标经过的轨迹实时显示为半透明路径预览线
3. **松手** → 路径确认，变为淡色虚线，车辆开始沿路径行驶
4. **车辆通过后** → 路径自动消失
5. **重新规划** → 随时可以再次点击正在行驶的车，重新画路线（覆盖旧路线）

#### 路线规划的手感要求

- 画线过程要流畅，不能有延迟或卡顿
- 路径预览线用半透明颜色（跟车辆同色），让玩家清楚这是哪辆车的路线
- 已确认的路径用虚线显示，不遮挡其他车辆
- 如果路径经过其他车辆当前位置附近，显示黄色警告（提示可能碰撞）

### 三、收费口

- 共 5 个收费口，横向均匀排列在画面中央
- 每个收费口同时只能处理 **1 辆车**
- 车辆到达收费口后自动停下，头顶显示进度条（处理时间 2-4 秒，随机）
- 进度条走完后，车辆自动继续沿路线行驶
- 如果一辆车到达收费口时该口正在忙，车辆会停在口前排队等待——**排队的车会挡住后面的路，造成拥堵**
- 玩家应该通过画路线把车分散到不同的收费口，避免都挤在一个口

### 四、碰撞与失败

**Game Over 条件（满足任一即结束）：**

1. **碰撞**：任意两辆车的碰撞体积重叠 → 立即 Game Over
   - 碰撞瞬间：画面闪红，碰撞点显示爆炸动画，所有车辆停止
   - 显示"碰撞！游戏结束"

2. **超时拥堵**：任意一辆车在场内停留超过 30 秒未成功通过收费口 → Game Over
   - 超时车辆头顶的计时器变红并闪烁作为预警（25 秒时开始警告）
   - 显示"拥堵超时！游戏结束"

**Game Over 画面：**
- 显示本局成绩：通过车辆数、游戏时长、最高连续通过数
- 大按钮："再来一局"
- 小字："最高纪录：XX 辆"（localStorage 存储）

### 五、难度递增曲线

| 游戏时长 | 车辆生成间隔 | 同时在场车辆上限 | 车速 | 新增元素 |
|---------|------------|----------------|------|---------|
| 0-30秒 | 每 5 秒 1 辆 | 3 辆 | 慢 | 只有小轿车，单方向 |
| 30秒-1分钟 | 每 3.5 秒 1 辆 | 5 辆 | 中 | 出现双向车流 |
| 1-2分钟 | 每 2.5 秒 1 辆 | 8 辆 | 中快 | 出现货车 |
| 2-3分钟 | 每 2 秒 1 辆 | 10 辆 | 快 | 出现特种车 |
| 3分钟+ | 每 1.5 秒 1 辆 | 12 辆 | 很快 | 随机关闭 1 个收费口（维修中） |

这条曲线的目的：**前 30 秒是教学期，让玩家学会画线和基本操作；1 分钟后开始有压力；2 分钟后需要快速决策；3 分钟后是真正的考验。**

### 六、计分与 HUD

画面顶部常驻显示（不遮挡游戏区域）：

| 指标 | 含义 | 显示格式 |
|------|------|---------|
| 已通过 | 成功通过收费站的车辆总数 | 数字，每 +1 时有微动画 |
| 效率 | 每分钟通过的车辆数 | X.X /min |
| 在场 | 当前场内车辆数量 | 数字，接近上限时变红 |
| 时长 | 本局游戏已进行时间 | MM:SS |

## 视觉风格

- **俯视图**，类似 Google Maps 的卫星视角
- **配色干净**：浅灰色路面、白色车道线、深色收费口建筑
- **车辆用简单矩形表示**，不同颜色区分类型，有圆角
- **路径线**：画线时半透明彩色，确认后变虚线
- **背景**：浅米色或浅灰色，不花哨
- **字体**：系统默认无衬线字体，加粗数字
- **整体感觉**：像 airport.apunen.com 一样干净、克制、信息清晰

## 交互适配

- **桌面端**：鼠标点击车辆 → 按住拖动画线 → 松开确认
- **移动端**：手指触摸车辆 → 按住拖动画线 → 松开确认
- 画面自适应屏幕宽度（响应式 Canvas）
- 竖屏横屏都能玩（收费站布局跟随屏幕比例调整）

---

## 技术实现要求

- **单个 HTML 文件**，所有 CSS 和 JS 内联，不依赖任何外部库或 CDN
- 使用 **HTML5 Canvas** 渲染所有游戏画面
- 使用 **requestAnimationFrame** 做游戏主循环
- 碰撞检测用矩形/圆形包围盒即可，不需要精确像素碰撞
- 路线数据用点数组存储，车辆沿点数组插值移动
- localStorage 存最高分
- 代码结构清晰，主要模块：
  - 游戏状态管理（开始/运行/结束）
  - 车辆生成器（按难度曲线定时生成）
  - 车辆更新（沿路线移动、碰撞检测、超时检测）
  - 收费口逻辑（排队、处理、释放）
  - 输入处理（点击/触摸、画线、路径确认）
  - 渲染器（Canvas 绘制所有元素）
  - HUD（分数、时间、状态）

## 最重要的事

**做一个好玩的游戏，不是做一个能跑的 Demo。**

好玩 = 前 30 秒很轻松让玩家学会操作 + 1 分钟后开始紧张 + 2 分钟后手忙脚乱 + 失败时想说"再来一局"。

去体验 [airport.apunen.com](https://airport.apunen.com/)，理解它为什么让人停不下来，然后把**同样的节奏感和紧张感**复制到这个高速公路收费站场景里。

如果做出来的东西玩家点开 10 秒就关掉了，那就是失败的。
