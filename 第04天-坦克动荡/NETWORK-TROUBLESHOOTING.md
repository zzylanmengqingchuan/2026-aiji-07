# 客户端网络与防火墙排查

游戏现在把页面、三维引擎、接口和实时连接全部放在同一个游戏域名下。客户端只需要允许访问游戏域名的 HTTPS（443 端口），不再依赖境外字体或公共 CDN。

## 最快判断方法

在有问题的电脑打开：

`https://tank.xiaoxiaole.space/network-check.html`

检测页会分别检查：

1. 游戏服务接口；
2. 本站三维引擎文件；
3. WebSocket 实时对战连接。

如果检测页本身都打不开，问题在该电脑、局域网、DNS 或防火墙，不在游戏操作代码。

## Windows 防火墙

1. 暂时关闭防火墙测试一次；如果关闭后可以进入，即可确认是规则拦截。
2. 重新开启防火墙，不要长期裸奔。
3. 在“允许应用通过防火墙”中允许使用的浏览器访问“专用网络”和“公用网络”。
4. 如果单位安装了第三方安全软件，还需要在该软件中放行浏览器和 `tank.xiaoxiaole.space`。

## 校园网或公司网

必须允许：

- `tank.xiaoxiaole.space` 的 443 端口；
- HTTPS；
- 安全 WebSocket（`wss`）升级连接。

若普通网页检查通过、实时连接失败，通常是网关或安全软件阻断 WebSocket。可用手机热点复测：热点正常而原网络失败，就需要网络管理员放行 WebSocket，前端代码无法绕过单位防火墙策略。

## DNS 与证书

- 确认电脑日期和时间正确，否则 HTTPS 证书可能校验失败。
- 依次尝试无痕窗口、清理该域名缓存、更换公共 DNS。
- 不建议直接用服务器 IP 代替域名，因为 HTTPS 证书和安全 WebSocket都依赖域名。

## 服务端 502

如果所有电脑都出现 502，应登录腾讯云检查 Node 进程，而不是修改客户端防火墙：

```bash
curl -s http://127.0.0.1:3100/health
pm2 status
pm2 restart tank-trouble
pm2 logs tank-trouble --lines 100
```

正式环境建议启用 `pm2 startup` 和 `pm2 save`，避免服务器重启后游戏进程没有自动恢复。
