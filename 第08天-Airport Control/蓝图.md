## 一、产品概述

Airport Control 是一款**纯前端、零依赖的空中交通管制（ATC）画线引导游戏**，玩法继承经典游戏 Flight Control 的核心循环，并加入"落地飞机转离场"的持续压力机制：

> 飞机从地图边缘不断进场，玩家按住飞机拖出航线，引导它落到**同颜色**的跑道；落地的飞机停场后会自动转为出港航班排队起飞；两架空中飞机相撞即游戏结束。活得越久、起降越多，分数越高。

交付形态为**单个 `index.html` 文件**（约 43 KB，内联全部 CSS/JS），双击即可在任意现代浏览器中游玩，无需安装、无需构建、无需服务器。

## 二、目标用户与使用场景

| 维度 | 说明 |
|---|---|
| 目标用户 | 喜欢休闲益智 / 反应类小游戏的玩家；Flight Control 类游戏的老玩家 |
| 单局时长 | 1–10 分钟（难度 5 分钟爬满，之后拼生存） |
| 使用场景 | 浏览器碎片化娱乐；课堂演示"AI 辅助从零开发完整游戏"的成果 |
| 设备 | 桌面浏览器（鼠标拖拽）为主，兼容触屏 |

## 三、核心玩法循环

```
进场警告（2.4s 虚线圈倒计时）
   → 飞机进入空域，直线飞行
   → 玩家拖拽画航线（实时吸附反馈）
   → 落到同色跑道（+1 落地）
   → 滑入航站楼停场（4–8s）
   → 自动转离场：滑到跑道头排队
   → 每跑道 8s 放行间隔自动起飞（玩家不可控）
   → 飞出地图（+1 离场）
失败条件：两架空中飞机相撞 → 立即 Game Over
```

设计要点：

1. **唯一死法是撞机**——没被引导的飞机飞出地图只会静默消失，不惩罚玩家，降低挫败感。
2. **离场不可控**制造"可预测的拥堵"：玩家必须提前为起飞航迹规划空域，这是后期难度的主要来源。
3. **无尽模式 + 效率指标**（每分钟起降数 Pace），支撑反复挑战。

## 四、v1.0 功能清单（已全部实现）

### 4.1 生成与难度系统
- 飞机在地图边缘**内侧 24 单位**生成，航向朝向地图中心 ± 随机偏移，避免"出生即掉头"。
- 进场警告：虚线圆圈 + 机型色箭头 + 2.4 秒倒计时进度弧；到期前做 10 秒直线冲突预测，冲突则自动重摇位置。
- 难度曲线：`t = min(1, elapsed/300)`，生成间隔 5.2s → 1.4s（×0.85–1.15 随机），同屏上限 3 → 12 架。
- 生成器每次评估 5 个候选边缘/航向，选预测冲突最小的，并避免连续同边缘。

### 4.2 画线引导（核心交互）
- 按住飞机（拾取半径 6 单位）拖出航线，采样间隔 1.4 单位。
- 松手后路径管线：**抽稀（≥4.2 单位）→ 单趟拉普拉斯平滑 → 落地路径末端接三次贝塞尔对齐跑道方向**，飞机平滑转弯进近而非硬折线。
- 逻辑航向对齐切线，**视觉航向指数平滑 + 转弯压坡度**，无瞬间掉头。
- 路径渲染：拖拽中 = 滚动虚线；已生效 = 机型色半透明实线。

### 4.3 跑道吸附与反馈
- 吸附判定半径恒为 10 单位，以路径最后一个采样点判定。
- 拖拽中实时反馈：进入吸附范围 → 路径变为跑道色并脉动，threshold 显示**可见吸附圈**；错色 / 错方向 → 红色脉动 + 飘字提示。
- 松手后路径强制吸附到跑道入口圆心——"进圈就算数"。
- 判定优先级：同色入口（eligible）→ 跑道错误端（wrong direction）→ 异色入口（wrong color）。

### 4.4 起降与离场系统
- 三条跑道（yellow 水平 / blue、red 垂直），各有进近方向限制，只能落同色跑道。
- 落地：沿跑道减速滑跑 → 滑行道 → 停机位（14 个停机位，停场 4–8s 随机）。
- 离场：自动滑出到跑道头排队（holding），每条跑道独立 8 秒放行间隔；起飞前做冲突预测，冲突自动延迟；HUD 显示各跑道放行倒计时与排队长度。
- 三机型：yellow（7 单位/秒，半径 2.4）/ blue（8，2.6）/ red（9，2.8），生成权重 3:3:2。

### 4.5 视觉区分与表现
- **起飞/地面保障飞机整机变灰**（机身 `#a7afb3`），进场飞机保持机型色——起降状态一眼可辨。
- 地面飞机缩小至 54% 且贴地，起飞平滑恢复；空中飞机带呼吸浮动；离场路线为白色滚动虚线。
- 原创俯视飞机模型：渐变机身 + 后掠主翼 + 尾翼 + 垂尾 + 驾驶舱 + 轮廓阴影。
- 碰撞预警：两机距离 ≤ 半径和 ×3.2 时显示白色脉动圆环；撞机触发爆炸特效。

### 4.6 HUD 与操控
- 统计板：Landings / Departures / Elapsed / Pace / 在场飞机数 / 历史最佳（可折叠）。
- 新手引导（"Drag this plane" / "Match here"），首次落地后消失。
- 空格暂停（失焦自动暂停）、M 静音；WebAudio 合成音效（落地 / 起飞 / 撞机），零音频资源文件。
- localStorage 持久化个人最佳成绩。

## 五、系统架构

```
┌─────────────────────── index.html（单文件）───────────────────────┐
│  UI 层    Canvas 渲染（世界/跑道/飞机/路径/特效） + DOM HUD 覆盖层  │
│  交互层   Pointer Events → 拾取 / 拖拽采样 / 吸附判定 / 键盘快捷键  │
│  逻辑层   主循环 rAF + dt                                          │
│           ├─ Spawner（警告 → 生成 → 冲突重摇）                     │
│           ├─ PathEngine（采样 → 抽稀 → 平滑 → 贝塞尔对齐）         │
│           ├─ PlaneFSM（warning→flying→landing→taxiin→parked        │
│           │            →taxiout→holding→departing）                │
│           ├─ RunwayManager（放行间隔 / 队列 / 起飞冲突延迟）       │
│           └─ CollisionSystem（预警环 / 撞机判定）                  │
│  资源层   WebAudio 合成音效、localStorage 最佳成绩                 │
└────────────────────────────────────────────────────────────────────┘
```

- 虚拟世界坐标 120×80 单位，按窗口等比缩放居中（letterbox），resize / 高分屏自适应。
- 全部美术为 Canvas 2D 矢量绘制，全部代码原创，零第三方依赖、零外部资源。

## 六、技术选型与理由

| 选择 | 理由 |
|---|---|
| 单文件 HTML | 作业/演示分发成本为零，双击即玩 |
| Canvas 2D（而非 WebGL/Three.js） | 俯视角下表现力足够，实现与调试成本低，性能余量大 |
| 原生 JS（无框架） | 游戏循环与状态机用框架反而增加复杂度 |
| WebAudio 合成音效 | 免音频文件，保持单文件交付 |

## 七、验证情况

- 脚本语法检查通过；无头 Chrome 冒烟：
  - 214 秒无引导长跑：正常生成、撞机触发 Game Over，无运行时错误；
  - 全生命周期自动化测试：flying→landing→taxiin→parked→taxiout→holding→departing 完整走通，落地/离场各 +1；
  - 行为断言：生成点距边缘 ≥20 单位（50/50）、航向朝中心（98%）、吸附后路径末点精确等于跑道入口、平滑后最大转角下降 65%。
