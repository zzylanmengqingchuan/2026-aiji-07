# 腾讯云 SSH 密钥免密登录：两种方式的区别与踩坑实录

> 作者：kimi ｜ 安吉集训 Day09 过程记录（2026-07-24）
>
> 起因是想给游戏开放端口，顺手研究了腾讯云轻量服务器的「SSH 密钥」功能。
> 最大的坑：**控制台创建密钥时私钥是浏览器自动下载的，没有"下载按钮"，事后也无法再次下载。**
> 最终结论：本机生成密钥（Day06 老办法）和控制台生成密钥（新办法）两种方式**同时有效**，两把钥匙开同一把锁。

## 背景

服务器：腾讯云轻量应用服务器（广州），Ubuntu 系统，公网 IP `175.178.106.164`，登录用户 `ubuntu`。

此前 Day06 已经用过一种免密登录方式（见 `day06-自动部署之路.md`）：本机 `ssh-keygen` 生成密钥对，手动把公钥装进服务器的 `~/.ssh/authorized_keys`。

今天（Day09）在控制台发现腾讯云也有「SSH 密钥」功能（左侧导航 → SSH 密钥），于是创建了名为 `test` 的密钥并绑定到实例，结果卡在了一个问题上：**公钥生成之后，私钥在哪下载？**

## 踩坑：找不到的"下载按钮"

在控制台的密钥列表里翻来翻去，操作列只有「绑定实例 / 解绑实例 / 编辑标签 / 删除」，**没有任何下载入口**。

查腾讯云官方文档（轻量应用服务器 · 管理密钥）才知道，这是刻意设计：

> 创建完成后将**自动下载**私钥，腾讯云不会保管您的私钥信息。私钥**仅有一次下载机会**，并将自动保存至您浏览器的默认下载地址。

也就是说：

- 点「确定」创建密钥的那一刻，浏览器**自动**把私钥 `.pem` 文件下载到了 `~/Downloads`，没有按钮、没有提示，体验上就像"没有下载入口"。
- 腾讯云服务器上**只存公钥，不存私钥**。事后控制台永远不会有下载按钮，丢了只能删除重建。

回 `~/Downloads` 一翻，`test.pem` 果然躺在那里——创建时间 13:06，和控制台里密钥的创建时间分秒不差。它当时就下载成功了，只是没人告诉我。

> 对比：阿里云控制台是创建后弹窗里有一个明显的「下载私钥」按钮；腾讯云是直接触发浏览器自动下载。上午老师用阿里云演示过，所以本能地在找那个按钮。

## 补救：把私钥安置到 ~/.ssh

```bash
cp ~/Downloads/test.pem ~/.ssh/test.pem
chmod 600 ~/.ssh/test.pem   # 权限必须是 600，否则 ssh 拒绝使用
```

验证登录成功：

```bash
ssh -i ~/.ssh/test.pem ubuntu@175.178.106.164 'whoami'
# 输出:ubuntu
```

## 两种方式的区别（大白话版）

SSH 密钥的本质是**一把锁配一把钥匙**：公钥是锁，装在服务器上，不怕人看；私钥是钥匙，留在自己电脑上，丢了别人就能进你家。

两种方式用的是**同一套机制，登录效果完全一样**，区别只在：这副锁和钥匙是在哪儿配出来的。

### 方式一：本机生成（Day06 老办法）

**你在自己家里配钥匙。** 本机 `ssh-keygen` 配出锁和钥匙，钥匙从诞生起就没离开过你的电脑，然后把锁快递到服务器（手动粘贴进 `authorized_keys`）。

### 方式二：控制台生成（今天新办法）

**腾讯云帮你配钥匙再寄给你。** 在网页点「创建密钥」，是腾讯云的服务器配出了锁和钥匙（RSA 2048）。锁它帮你装到门上（「绑定实例」按钮），钥匙塞给你就不管了（创建瞬间浏览器自动下载 `.pem`，只此一次）。

### 对比表

| | 本机生成（方式一） | 控制台生成（方式二） |
| --- | --- | --- |
| 密钥对在哪里生成 | 自己的电脑（如 ed25519） | 腾讯云的服务器（RSA 2048） |
| 私钥出过门吗 | 没有，最安全 | 从云端经网络下载到本机，多一次暴露机会 |
| 公钥怎么上服务器 | 手动装进 `authorized_keys`（得先能登录） | 控制台「绑定实例」自动注入 |
| 私钥丢了怎么办 | 重新生成一对，重装公钥 | 无法找回，只能删除密钥重建 |
| 适合场景 | 已经能登录服务器 | 还没有任何登录手段的冷启动 |

**为什么腾讯云只给一次下载机会？** 它故意不存私钥——它要是存了，自己被攻破那天，所有用户的私钥就全泄露了。"只存锁、不存钥匙"是对用户负责。

**安全圈最佳实践是方式一（本机生成）**：私钥全程不碰网络。方式二的好处是省心，不用自己往服务器装公钥，适合冷启动。

## 关键认知：两种方式可以同时有效

服务器的 `authorized_keys` 里可以放**多把公钥**，任何一把对应的私钥都能登录同一个账号。这次实测验证：

```bash
# 方式一:本机默认钥匙(day06 手动装的公钥,至今有效)
ssh ubuntu@175.178.106.164 'whoami'        # ✅ ubuntu

# 方式二:控制台密钥 test.pem
ssh -i ~/.ssh/test.pem ubuntu@175.178.106.164 'whoami'   # ✅ ubuntu
```

控制台绑定新密钥是在 `authorized_keys` 里**追加**一行公钥，不会覆盖之前手动装的。所以现在服务器上装着两把锁，手里两把钥匙都能开门，互为备份。

## 这台服务器现在的三条登录路径

1. **控制台网页终端**（OrcaTerm / VNC）：一直能用，不依赖本机密钥，救急专用。之前"没下载 pem 也能登录"走的就是这条路——认证发生在腾讯云后端，和本机有没有私钥无关。
2. **本机 ssh + 默认钥匙 `id_ed25519`**：方式一，已验证可用。
3. **本机 ssh + `test.pem`**：方式二，已验证可用。

注意：绑定密钥后，腾讯云默认**禁止 root 的密码登录**（安全设计）。密码登录这条路基本关了，所以钥匙务必保管好。

## 关于"把密钥交给 AI"

一个容易误解的点：不需要把密钥文件"发送"给某个云端 AI。Agent 就跑在这台 Mac 上，它执行 `ssh` 用的就是本机 `~/.ssh/` 里的密钥，和人自己敲命令完全等价。密钥放进 `~/.ssh/` 那一刻，人和 Agent 都能用了——所谓"让 Agent 能登录服务器"，本质就是"让这台电脑的 ssh 能免密登录"。

## 复盘

- 这次最大的困惑来自一个静默行为：**自动下载没有提示**。以后在任何云平台创建密钥，第一反应应该是去浏览器的默认下载目录找 `.pem`。
- 网页终端能登录 ≠ 本机 ssh 能登录，这是两条独立的路，别混为一谈。
- "能不能登录"这种问题，实测一把比推测十句都管用。
- 至此免密链路全部打通：改代码 → rsync/部署脚本 → ssh 免密执行 → 外部验证，Agent 可以全程代劳（延续 Day06 的自动化路线）。
