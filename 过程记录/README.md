# 过程记录文件夹 · 提交规范（人 & Agent 必读）

本文件夹存放安吉集训期间的过程记录、经验文章，会持续更新。提交前必须遵守以下规范。

## 文件命名

```
dayNN-主题.md
```

- `NN` 为集训天数（两位数字），如 `day06-自动部署之路.md`
- 天数不确定时，先问用户当天是第几天，禁止自行猜测
- 现有项目文件夹沿用 `第XX天-项目名` 的旧约定不动（如 `第04天-坦克动荡/`），过程记录统一用 `dayNN-` 前缀

## 文章头部署名

每篇文章顶部必须有一行署名块：

```markdown
> 作者：kimi / codex / grok … ｜ 安吉集训 DayNN 过程记录
```

"作者"写**实际产出该内容的 agent 名**（多 agent 混用，靠这行溯源）。

## commit 规范

- message 前缀写清天数与主题，如：`day06 过程记录：自动部署之路`
- **author 必须是仓库主人**：`zzylanmengqingchuan <3258580458@qq.com>`（本地 .gh-sync 已写入 git config，不得更改）
- 禁止编造 `xx@users.noreply.github.com` 邮箱（会被 GitHub 关联到陌生真实用户）
- main 分支受保护，禁止 force push（改写历史需用户明确批准并走临时解除流程）
